# Why NOC Living Docs?

## 为什么需要 NOC Living Docs？

AI Agent 写代码很快，但项目记忆很容易丢。

常见问题是：

- 需求留在聊天记录里
- 当前实现没人记录
- 测试结果没有沉淀
- 约束和禁区被忘掉
- 新的 Codex 会话会误判项目现状

NOC 的做法很简单：给每个功能一个固定的文档位置，让需求、现状、测试、限制和变更记录跟着代码一起维护。

## The problem

AI agents can change code quickly, but project memory is often lost.

## The failure mode

- requirements stay in chat
- current behavior is undocumented
- tests are not recorded
- guardrails are forgotten
- new Codex sessions hallucinate project status

## The solution

NOC gives every feature a stable documentation home.
